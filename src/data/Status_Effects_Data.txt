# Status_Effects_Data.txt
#
# File contains information on "display" status effects which are shown to the user
# as well as "internal" status effects, which are used to represent components of display status effects
# and are used to actually modify the properties of the monsters/players they affect.
#
# If a non-zero magnitude is supplied to the Combat.addStatus method, it overrides any zero 
# value declared here. If the magnitude declared here is 0, it probably indicates the magnitude is intended 
# to be supplied to (and overridden by) the calling method.  If the magnitude is negative, it indicates that 
# the absolute value of the number provided should be multiplied by the provided magnitude from the upper layer.
#
# The IMMUNE tag is applied when the status is an internal debuff.  The IMMUNE tag is not implemented as of this writing.
# There are three settings for this tag:
#   None (implied when no tag is present) -- all monsters can be affected
#   Boss -- only extremely powerful monsters ignore this effect completely
#   Strong -- somewhat powerful monsters (and bosses) ignore this effect completely
#
# The CATEGORY tag will assign category to the ability to indicate things such as
# whether an ability is considered a buff or debuff or a spellsword enchantment etc.
#
# The RECURRING tag uses either True or False to determine if this Internal property needs to be reapplied 
# during the upkeep phase of each turn.
#
# Any other confusing tags are likely an artifact of an older implementation strategy for status effect parsing and 
# interpretation that is no longer in use.  Tags that are currently in use or that will eventually be used are:
#
# NAME, TYPE, RECURRING, IMMUNE, IMAGE, ELEMENT, INTERNAL, MAGNITUDE, CATEGORY
# Of those, only IMMUNE has not yet been in use.
